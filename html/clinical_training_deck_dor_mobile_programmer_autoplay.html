Programming the DOR Table

A Guide for Clinical SAS Programmers

📊

What is DOR?

Duration of Response (DOR) is a critical efficacy endpoint in oncology trials.

It measures the time from the first documented response (Complete Response [CR] or Partial Response [PR]) until:

  • Disease Progression (PD)
  • Death from any cause

Unlike Progression-Free Survival (PFS), DOR is calculated only for the subset of patients who actually responded to the treatment (e.g., Investigational Drug A).

Key Terminology

  • Event: The patient experienced progression or death. (CNSR = 0)
  • Censored: The patient has not progressed/died by their last assessment. (CNSR = 1)
  • + (Plus Sign): Used in tables to denote a censored observation (e.g., a DOR of 12.5+ months).
  • NR (Not Reached): Occurs when the Kaplan-Meier curve does not drop below 50%. The median DOR cannot be estimated yet.

The 3 Denominators Trap

Warning: The most common QC finding in DOR tables is using the wrong denominator.

You must manage three distinct N's:

  1. Study/Arm N: The total Evaluable or ITT population (from ADSL). Usually shown in the column header.
  2. Responder N: The total number of responders. This is the denominator for your Kaplan-Meier analysis.
  3. Censored N: The total number of censored patients. This is the denominator when calculating percentages for "Reasons for Censoring".

Standard vs Reverse KM

We run Kaplan-Meier (PROC LIFETEST) twice for a complete DOR table.

1. Standard KM:

  • Estimates the time to Event (Progression/Death).
  • Yields Median DOR, 95% CI, and landmark event-free rates.

2. Reverse KM:

  • Estimates the Median Follow-Up time for the responder cohort.
  • We artificially swap the Event and Censor flags. Progressions become censored; active follow-ups become events.

SAS Logic: CDISC Setup

First, identify responders from ADRS (Tumor Response) for Investigational Drug A.

SAS Data Step
/* 1. Find Responders (CR or PR) */ data responders; set adrs; where paramcd='OVRSG' and avalc in ('CR', 'PR') and trta = 'Drug A'; keep usubjid; run; /* 2. Subset ADTTE (Time to Event) */ data dor_tte; merge responders(in=a) adtte(where=(paramcd='DOR')); by usubjid; if a; /* Keep ONLY responders */ run;

SAS Logic: Appending '+'

When reporting the Min and Max DOR, you must append a '+' if that specific observation was censored.

SAS Data Step
/* Sort to find Min/Max */ proc sort data=dor_tte; by trta aval; run; /* Format value as character */ data dor_fmt; set dor_tte; if cnsr = 1 then aval_c = strip(put(aval, 8.1)) || '+'; else aval_c = strip(put(aval, 8.1)); run;

SAS Logic: Reverse KM

To calculate Median Follow-up for Responders, flip the censoring variable.

SAS Proc Lifetest
/* Flip the Censor Flag */ data rev_km; set dor_tte; rev_cnsr = 1 - cnsr; /* 0 becomes 1, 1 becomes 0 */ run; /* Run Lifetest for Follow-up */ proc lifetest data=rev_km; time aval * rev_cnsr(1); strata trta; ods output Quartiles=fup_med; run;

SAS Logic: Landmarks

Tables often ask for the probability of remaining in response at specific timepoints (e.g., 6 and 12 months).

SAS Proc Lifetest
/* Standard KM with Timelist */ proc lifetest data=dor_tte timelist=6 12 reduceout; time aval * cnsr(1); strata trta; ods output ProductLimitEstimates=ple; run; /* 'ple' dataset contains Survival estimates at exactly 6 and 12 */

QC Checklist ✅

  • Population: Are non-responders strictly excluded from the Lifetest?
  • Denominators: Does the N for "Reasons for Censoring" equal the total number of censored responders?
  • Min/Max: Are the '+' signs accurately reflecting the cnsr flag of the specific min and max records?
  • Follow-up: Did you remember to use the Reverse KM method (flipping the censor flag) for the follow-up median?
  • NR formatting: Is a missing median properly formatted as "NR" or "NE" as per the SAP?
